# Notify2Discord 共通開発ルール

## バージョン運用
- Codex が修正を行うたびに `app/build.gradle.kts` の `versionName` を `+0.0.1` する（例: `1.1.0 -> 1.1.1`）。
- 同時に `versionCode` を必ず `+1` する。

## 実装・品質
- 変更後は可能な限り `./gradlew :app:assembleDebug` でビルド確認を行う。
- ユーザー向け文言は技術用語を避け、原因と対処が分かる表現を優先する。
- Discord通知は「短い本文 + 見やすいEmbed」を基本とし、情報の重複を避ける。

## 設定互換性
- DataStoreのキーや保存形式を変更する場合は後方互換を維持する。
- 既存ユーザー設定は上書きせず、必要なときのみ限定的な移行処理を入れる。
- バックアップ/復元に関わる仕様変更時は `schemaVersion` の扱いを明示する。
